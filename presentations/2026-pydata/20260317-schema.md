# Prototype Data Schema

## Bar Chart

```python
spread("lake", dir="x")
rect(h="count")
```

means that we have a discrete key `lake` (b/c it's being used in a spread) and we have a value
`count` that is continuous (b/c it's mapped to height)

### Afforded queries

- greatest, least, pairwise

## Stacked Bar Chart

```python
chart(data).flow(
    spread("lake", dir="x"),
    stack("species", dir="y"),
).mark(
    rect(h="count", fill="species")
)
```

key: `lake: discrete -> species: discrete (part-to-whole w/ lake)`
value: `count: continuous, species: discrete`

### Afforded queries

- greatest, least, pairwise on lake totals
- (general) part-to-whole comparisons
- hard to read off exact values without labels (compare to grouped bar chart where it's much easier
  to make precise readings and comparisons)
- general trends of species in each lake

## Sorted Bar Chart

```python
chart(data).flow(
    spread("lake", dir="x"),
    derive(lambda d: sorted(d, key="count")),
    stack("species", dir="y"),
).mark(
    rect(h="count", fill="species")
)
```

key: `lake: discrete -> species: ordered (part-to-whole w/ lake)`
value: `count: continuous, species: discrete`

### Afforded queries

Same as before, but now it's easy to read the biggest, smallest, second biggest, second smallest,
etc. entry in each lake.

## Ribbon Chart

```python
	Layer([
  chart(data).flow(
    spread("lake", dir="x", spacing=64),
    derive(lambda d: sorted(d, key="count")),
    stack("species", dir="y"),
  ).mark(
    rect(h="count", fill="species").name("bars")
  ),
  chart(select("bars"))
    .flow(group("species"))
    .mark(area(opacity=0.8)),
])
```

key: `lake: discrete -> species: ordered (part-to-whole w/ lake)`
value: `count: continuous, species: discrete`

key: `species: contiguous` (b/c `area` links them)

### Afforded queries

Same as before, but now it also affords reading trends of species across the lakes. (In fact, we
know from Gestalt literature that connection is a stronger principle than proximity so these trends
dominate the within-lake comparisons although they are still possible!)

## Polar Ribbon Chart

Basically the same as ribbon chart, but it's less accurate.

### Afforded queries

Pretty similar, but things become less accurate. The Gestalt structure is quite similar.

## Highlighted Polar Ribbon Chart

Specific queries on the data structure are much easier and actually others are not if they are gray b/c
they become indistinguishable!

### Afforded queries

Highlighted species trends are afforded over everything else.

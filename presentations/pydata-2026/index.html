Welcome! 30 min talk + 15 min Q&A. We'll go from "why another viz library" to live composable charts, ending with scatter plots where every point is a pie chart. Let's start with a problem.
0:00 — The Problem

The Visualization Ceiling

① Bar chart — easy everywhere plt.bar(...)
② Stacked bar — still fine df.plot.bar(stacked=True)
③ Stacked bar + line overlay — getting awkward ax.twinx()...
④ Scatter where each point is a pie — now what?
Every library has a ceiling. The first three are fine in matplotlib/seaborn/plotly. But once you need real composition — overlays, glyphs, nested charts — you're rewriting everything. 4 min mark.
The Idea

Charts as Composable Data Structures

Most libraries treat charts as drawing commands.
GoFish treats them as data transformations.

Consequence: charts compose the same way functions compose — stack one inside another, swap coordinates, reference layers by name.
This is the one-sentence pitch. Once you accept it, everything else follows — nesting, coordinate transforms, layer references all fall out naturally.

4:30 — The Flow API

Basic Bar Chart

from gofish import chart, spread, rect

chart(data).flow(
    spread("lake", dir="x")
).mark(
    rect(h="count")
).render(w=500, h=350)

chart(data) — attach data to a builder

.flow() — pipe through layout operators

spread("lake", dir="x") — one column per lake

.mark(rect(h="count")) — height encodes count

The three-part structure: chart → flow → mark. This is the whole API in miniature. Notice there are no x/y coordinates — GoFish infers them from the operator.
6:30 — Composition = Adding Lines

Add Stacking

chart(data).flow(
    spread("lake", dir="x"),
    stack("species", dir="y"),  # ← new
).mark(
    rect(h="count", fill="species")
).render(w=500, h=350)
One new line → stacked chart.
No rewriting. No coordinate math.
Key message: composition = adding lines, not rewriting. The spread and stack operators are independent — they each do one thing and compose cleanly. 8 min mark.
9:00 — Grammar of Graphics

The Pattern Has a Name

Data
Your DataFrame — the source of truth
Operators
How to arrange: spread, stack, scatter, group
Marks
What to draw: rect, circle, line, area
Channels
Data → visual: h="count", fill="species"
Coordinates
What geometry: linear (default), clock()
Why this matters: the abstraction, not the library, explains what's possible. The ceiling moves when the algebra is right.
Don't lecture. Name the pattern they already saw. The point is: once you have these five things, coordinate transforms fall out naturally. 10 min mark.
11:00 — One-Line Coordinate Swap

Bar → Pie in One Line

from gofish import clock

chart(data, options={"coord": clock()}).flow(
    stack("species", dir="x")
).mark(
    rect(w="count", fill="species")
).render(w=400, h=400)

Same operators, same marks.

Only coord=clock() changed.

The geometry wraps x into angle, y into radius.

This is where the abstraction pays off. Bar chart and pie chart are the same thing in different coordinate systems. 13 min mark.
13:00 — Layer Selection

The Overlay Problem

In matplotlib, drawing a line through scatter points means manually aligning coordinates: 

fig, ax = plt.subplots()
ax.scatter(df["miles"], df["gas"], c=df["year"])
# Now sort and connect — by hand
sorted_df = df.sort_values("year")
ax.plot(sorted_df["miles"], sorted_df["gas"], "k-", lw=0.8)
GoFish alternative: name a layer, then select it.
The second chart inherits all layout decisions from the first.
The problem: once you have two separate chart objects, nothing links their coordinate spaces. GoFish solves this with named layers and select(). 14 min mark.
14:30 — select()

Connected Scatter via select()

from gofish import Layer, select, circle, line

Layer([
    chart(data)
        .flow(scatter("year", x="miles", y="gas"))
        .mark(circle(r=4).name("points")),

    chart(select("points"))
        .mark(line(stroke="black")),
])

.name("points") — registers each node by name

select("points") — retrieves them in a new chart

The line inherits the scatter's x/y positions — no coordinate math.

Name a layer, reference it. The second chart knows nothing about miles or gas — it just connects the already-placed nodes. 16 min mark.
16:30 — scaffold + select

Stacked Area Chart

from gofish import scaffold, area, group

Layer([
    chart(data).flow(
        spread("lake", dir="x", spacing=64),
        stack("species", dir="y"),
    ).mark(
        scaffold(h="count", fill="species").name("bars")
    ),

    chart(select("bars"))
        .flow(group("species"))
        .mark(area(opacity=0.8)),
])

scaffold — positions without drawing (invisible)

select("bars") + group("species") — one area per species

scaffold is a "ghost mark" — it lays out positions but draws nothing. The area mark then connects those positions. 18 min mark.
18:00 — Glyphs

Charts Within Charts

A mark function can return an entire chart.

Normally: .mark(rect(...)) — mark returns a shape

Glyph: .mark(lambda data: chart(data[0]["collection"])...) — mark returns a chart

GoFish renders the inner chart into the bounding box allocated for each outer data point.

This works because GoFish builds an AST — a tree of nodes.
Nesting = child nodes. Same as HTML elements.
This is the moment of revelation. The mark API doesn't distinguish between a shape and a chart — both return a node tree. 20 min mark.
20:30 — The Showstopper

Scatter Where Each Point Is a Pie

chart(scatter_data).flow(
    scatter("lake", x="x", y="y")
).mark(
    lambda data:
        chart(
            data[0]["collection"],
            options={"coord": clock()}
        )
        .flow(stack("species", dir="x", h=20))
        .mark(rect(w="count", fill="species"))
)

Outer chart: positions lakes in space

Inner chart: renders a pie for each lake

data[0]["collection"] — per-lake species data

Walk through slowly. Outer scatter places the pies. Inner chart renders each pie independently. The coordinate transform runs per-glyph. Pause after showing the output — let it land. 23 min mark.
23:30 — Why It Works

The Nesting Ladder

① Bar: spread + rect
② Stacked bar: spread + stack + rect
③ Pie: stack + rect + clock()
④ Scatter-pie: scatter + nested(stack + rect + clock())
Each step adds one operator or one level of nesting.
The complexity scales with your intent, not with the library.
The progression from bar to scatter-pie is 4 small steps. That's what composition buys you. 24 min mark.
24:30 — Architecture

Why Composition Works: The AST

Frame (root)
│
Spread ×6 lakes
│
Stack ×5 species
│
Rect (leaf)

Three render passes over the tree:

① Domain inference — what data ranges exist?

② Layout calculation — how do elements fit?

③ Placement — final positions → SVG

GoFish doesn't draw anything imperatively. It builds a tree, then does three passes over it. Nesting is just adding a child node — the passes handle the rest. 25 min mark.
24:00 — Python API

Python Integration

from gofish import chart, spread, rect, derive
import numpy as np
import pandas as pd

data = pd.DataFrame({
    "lake": ["Lake A"] * 5 + ["Lake B"] * 5,
    "species": ["Bass","Trout","Catfish","Perch","Salmon"] * 2,
    "count": [23, 31, 29, 12, 8, 25, 34, 41, 21, 16],
})

chart(data).flow(
    derive(lambda df: df.assign(log_count=np.log(df["count"]))),
    spread("lake", dir="x"),
).mark(
    rect(h="log_count", fill="species")
).render(w=600, h=400)
derive() round-trips data back to Python — full pandas/numpy available.
The Python bindings let you use the full PyData ecosystem inside derive(). The chart spec is JSON, data is sent as Arrow. 26 min mark.
